# Sunlife CMS - Рабочий процесс

## Архитектура

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Админка (PHP) │────▶│  public/content/│────▶│   git push      │
│   на хостинге   │     │   *.json файлы  │     │   (изменения)   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
                                                        │
                                                        ▼
                                               ┌─────────────────┐
                                               │  Разработчик    │
                                               │  git pull       │
                                               └─────────────────┘
                                                        │
                    ┌───────────────────────────────────┼────────────────────────┐
                    │                                   ▼                        │
                    │  ┌─────────────────┐     ┌─────────────────┐              │
                    │  │ npm run build   │────▶│  dist/ (статика)│──────────────┼────▶ Хостинг
                    │  │                 │     │  HTML+CSS+JS    │              │      (production)
                    │  │ 1. sync-content │     │  (JSON в JS)    │              │
                    │  │ 2. tsc          │     └─────────────────┘              │
                    │  │ 3. vite build   │                                    │
                    │  │ 4. prerender    │                                    │
                    │  └─────────────────┘                                    │
                    │                                                       │
                    └───────────────────────────────────────────────────────┘
```

## Как это работает

### Для посетителя сайта
- Сайт 100% статичный — никаких HTTP-запросов за данными
- Все JSON-файлы встроены в JS-бандл при сборке (`npm run build`)
- Мгновенный рендер, никакого "мигания" контента
- Максимальный PageSpeed Insights

### Для редактора (через админку)
1. Открывает `https://sunlife-ufa.ru/admin/`
2. Входит с паролем `sunlife2025`
3. Редактирует цены, FAQ, отзывы и т.д.
4. Нажимает "Сохранить" — JSON файлы обновляются на сервере
5. Сообщает разработчику, что нужен rebuild

### Для разработчика (после изменений в админке)
```bash
# 1. Получить изменённые JSON с хостинга
git pull origin main

# 2. Скопировать JSON из public/content/ в src/content/
# (это делает npm run sync-content автоматически)

# 3. Собрать проект
npm run build

# 4. Загрузить dist/ на хостинг
# (через FTP, rsync, или CI/CD)
```

## Структура JSON файлов

| Файл | Что содержит | Редактор |
|------|-------------|----------|
| `pricing.json` | 6 пакетов услуг | Цены |
| `faq.json` | 12+12 FAQ (клиенты + партнёры) | FAQ |
| `gallery.json` | 8 фото галереи | Галерея |
| `benefits.json` | 4 преимущества | Преимущества |
| `reviews.json` | 8 отзывов | Отзывы |
| `steps.json` | 5 шагов + вводный текст | Почему мы |
| `geography.json` | 7 городов + текст | География |
| `partnership.json` | О нас, предложения, цены, команда, проекты | Партнёрство |
| `meta.json` | SEO: title, description, OG для 4 страниц | SEO |

## Скрипты

```bash
npm run dev          # Dev сервер (localhost:5174)
npm run sync-content # Копировать public/content/*.json → src/content/
npm run build        # Полная сборка (sync + tsc + vite + prerender)
npm run preview      # Просмотр production сборки
```

## Безопасность

- Админка защищена паролем (bcrypt hash в `api/index.php`)
- Пароль: `sunlife2025` (сменить hash в `public/admin/api/index.php`)
- Автоматические бэкапы JSON при сохранении (10 последних)
- Валидация имён файлов (только [a-zA-Z0-9_-])
- Проверка JSON перед записью

## Требования к хостингу

- PHP 8.3+ с GD (для обработки изображений)
- Права на запись в `public/content/` и `public/images/cms/`
- .htaccess для PHP (если Apache)
